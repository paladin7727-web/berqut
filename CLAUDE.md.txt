## TECHNICAL REQUIREMENTS

### Visual & Design
- **Intro Animation**: Full-screen 3D animation (WebGL / Three.js) on page load
  - Dynamic visual sequence with rendering of company objects/scenes
  - Animation completes with BERQUT logo centered on screen (golden color)
  - Logo is clickable trigger — click opens main website
  - Intro shows once per session (sessionStorage), returning visitors go straight to main site
  - Mobile/weak devices: simplified version (video loop instead of full WebGL)
  - Must respect prefers-reduced-motion and include "Skip intro" button

- **Color Scheme**:
  - Black background everywhere (required)
  - Single accent color: warm bronze-gold (eagle/Berqut symbolism)
  - Premium typography (serif for headlines, neutral sans for body text)

### Multilingual & SEO (HIGHEST PRIORITY)
- **7 Languages**: Russian, Kazakh, English, German, Chinese, Spanish, Italian
- **Architecture**: Language folders, not subdomains (/ru/, /en/, /kz/, /de/, /zh/, /es/, /it/ on main domain)
- **hreflang tags**: Correct hreflang between all language versions
- **Sitemap**: Multilingual sitemap.xml for each language
- **Schema.org markup**: RealEstateAgent, LocalBusiness on every language version
- **Core Web Vitals**: Optimize for LCP, FID, CLS (3D assets must not block page load)
- **SEO Content**: Localized meta titles/descriptions optimized for each geo-market (not just translations)

### Domain Architecture
- **Main domain**: BERQUT.COM (primary)
- **Secondary domains**: BERQUTGROUP.COM, BERQUT-GROUP.COM
- **Redirect strategy**: 301 server-level redirects from secondary to main (preserve SEO weight)

### Performance & Technical
- **Framework**: Next.js (SSR/SSG, native i18n routing, SEO-friendly)
- **3D Stack**: react-three-fiber / Three.js + GSAP for animations
- **i18n**: next-i18next or similar for clean URL structure
- **Mobile-first**: Fully responsive, tested on mobile
- **AI Chatbot** (MVP): Multilingual LLM-based bot (Phase 2), responds to questions about services
- **Loading**: Critical content (text, SEO markup) renders independently from WebGL (lazy-load Three.js)
